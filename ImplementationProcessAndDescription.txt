Порядок действий:
1. Установил локально виртуальную машину Ubuntu Server (сервер 1).
2. Клонировал её (сервер 2).
3. Установил apache http. (сервер 1)
4. Установил a2enmod cgi. (сервер 1)
5. Сконфигурировал apache, для запуска cgi скриптов, не только с директории cgi-bin, но и с корневого каталога apache (var/www/) (apache2.conf) (сервер 1)
6. Из под пользователя www-data, сгенерировал публичный ключ и установил его на сервер 2. (сервер 1)
7. Создал две облачные машины
8. Открыл порты:
Cервер 1: 22(ssh), 80(http)
Cервер 2: 22(ssh)
Повторил 3-6 действия уже на облачных машинах.

Для сохранения корректного имени файла необходимо запустить wget с параметром --content-disposition.
Например: wget --content-disposition 52.142.9.144/getFile?patch=/bin/bash

Внешний ip адрес машины (cервер №1 - ubuntu): 52.142.9.144

getFile #cgi shell скрипт  посредством ssh получает заданный файл с сервера 2 и отдает его на клиентское устройство.

getFileLocal #cgi shell отдает заданный файл сервера 1  на клиентское устройство.

apache2.conf #конфиг файл apache
+=|<Directory /var/www/>
	
+=|Options ExecCGI
	
+=|AllowOverride None
	
+=|SetHandler cgi-script
	
+=|Require all granted

+=|</Directory>
